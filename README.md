- **Resposta 05**
No contexto dos concursos para os Tribunais Regionais do Trabalho (TRTs), a stack de desenvolvimento em Java é o pilar central, especialmente devido à migração para a Arquitetura PDPJ-Br (Plataforma Digital do Poder Judiciário Brasileiro)
.
Com base nos editais recentes e no padrão de cobrança da FCC, os tópicos com maior probabilidade de aparecerem na sua prova são:

1. Framework Spring (O "Coração" da Stack)
A FCC tem uma "tara" por cobrar as funcionalidades e, principalmente, as anotações do ecossistema Spring
.
Spring Boot: Entenda o conceito de Fat JAR (arquivo autônomo que embarca o servidor web, como Tomcat ou Jetty)
. Decore as anotações básicas como @SpringBootApplication, @RestController, @Service e @Autowired
.
Spring Cloud e Microsserviços: Este é o tema mais quente
. Você deve dominar o Netflix Eureka (Service Discovery) — inclusive sua porta padrão 8761 — e o Zuul ou Spring Cloud Gateway (API Gateway)
. O padrão Circuit Breaker (resiliência) também é recorrente
.
2. Persistência de Dados (JPA e Hibernate)
A integração do Java com bancos de dados relacionais é cobrada via JPA 2.0 e Hibernate
.
Mapeamento Objeto-Relacional (ORM): Foco total em anotações de mapeamento como @Entity, @Table, @Id, @OneToMany e @JoinColumn
.
Hibernate Envers: Ferramenta para auditoria de dados. A FCC costuma perguntar sobre a anotação @Audited para rastrear mudanças nos registros
.
Flyway: Biblioteca essencial para versionamento do esquema do banco de dados. Saiba que ela utiliza a tabela flyway_schema_history para controle
.
3. Arquitetura de Microsserviços e APIs
A stack Java nos tribunais não é mais monolítica
.
RESTful APIs: Domine as constraints do REST (Stateless, Layered System, Client-Server) e os verbos HTTP (GET, POST, PUT, DELETE e especialmente o PATCH para atualizações parciais no contexto da PDPJ)
.
JSON e JWT: O uso de JSON como formato de intercâmbio e JSON Web Tokens (JWT) para autenticação e autorização via OAuth2/Keycloak é regra na arquitetura atual
.
4. Engenharia de Software e Qualidade
Testes Automatizados: Foco em JUnit para testes unitários e Mockito para criação de mocks
. Entenda a diferença entre um Mock e um Spy (o Spy é um objeto real até que se prove o contrário)
.
Padrões de Projeto (GoF): A FCC adora "historinhas" onde você deve identificar o padrão. Os favoritos são: Singleton (instância única), Adapter (compatibilizar interfaces), Observer (notificações) e Strategy (algoritmos intercambiáveis)
.
5. Tecnologias em "Desidratação"
Embora o CNJ recomende a "desidratação" (substituição progressiva) de tecnologias antigas, elas ainda podem constar no edital como conhecimento de base:
JSF (JavaServer Faces), EJB (Enterprise JavaBeans) e RichFaces
.
Dica do Especialista: A FCC é uma banca conceitual e "tizinha", que se apega a detalhes e literalidade das especificações
. Para a stack Java, não basta saber programar; é preciso saber o nome exato dos pacotes e das propriedades de configuração
.

Abaixo, apresento um glossário com os conceitos fundamentais para o cargo de Analista de TI, organizados pelas áreas de maior relevância identificadas nos editais e revisões técnicas.

1. Arquitetura e Desenvolvimento (PDPJ-Br)
API Gateway (Netflix Zuul): Funciona como um ponto único de entrada para as requisições em uma arquitetura de microsserviços, sendo responsável pelo roteamento dinâmico, segurança e monitoramento
.
Arquitetura PDPJ-Br: Plataforma Digital do Poder Judiciário Brasileiro, concebida como uma nuvem para integrar todos os sistemas do Judiciário em um ambiente unificado e comunitário
.
HATEOAS: Princípio do modelo RESTful que permite que uma API forneça links rastreáveis nos recursos devolvidos, indicando ao cliente quais ações podem ser realizadas a partir daquele estado
.
JSON Web Token (JWT): Padrão utilizado para a criação de tokens de autenticação e autorização, composto por três partes: cabeçalho (header), carga útil (payload) e assinatura
.
Keycloak: Ferramenta de código aberto utilizada para gerenciamento de identidade e acesso (IAM), provendo autenticação Single Sign-On (SSO) baseada em protocolos como OAuth2
.
Microsserviços: Estilo arquitetural onde a aplicação é dividida em serviços pequenos, independentes e descentralizados, que se comunicam via APIs leves
.
Service Discovery (Netflix Eureka): Atua como um catálogo de serviços onde cada instância de microsserviço se registra para ser descoberta por outros componentes da rede
.
Spring Boot: Extensão do ecossistema Spring que facilita a criação de aplicações Java autônomas e prontas para execução, utilizando o conceito de "convenção sobre configuração" e servidores web embarcados
.
Stateless: Característica obrigatória dos microsserviços na PDPJ-Br, onde o servidor não armazena informações sobre o estado do cliente entre as solicitações
.
2. Persistência e Bancos de Dados
Atributo Derivado: Propriedade de uma entidade que não é armazenada fisicamente, mas calculada a partir de outros atributos, como a "idade" calculada da "data de nascimento"
.
Data Warehouse: Repositório de dados integrado, não volátil e histórico, orientado por assunto para apoiar a tomada de decisão
.
Flyway: Biblioteca para versionamento de banco de dados que rastreia o estado do esquema através de uma tabela padrão denominada flyway_schema_history
.
Hibernate Envers: Módulo do framework Hibernate que provê auditoria automática de entidades Java, permitindo manter um histórico de alterações
.
JPA (Java Persistence API): Especificação Java para mapeamento objeto-relacional (ORM) que permite gerenciar dados relacionais em aplicações Java
.
Normalização: Processo de organização de tabelas para eliminar redundâncias e anomalias de inclusão, exclusão e modificação
.
OLAP (Online Analytical Processing): Tecnologia que permite a análise multidimensional de grandes volumes de dados através de operações como Roll-up (resumo) e Drill-down (detalhamento)
.
3. Infraestrutura, DevOps e Segurança
CI/CD: Coleção de práticas para integração contínua (automação de builds e testes) e entrega/implantação contínua (automação da liberação para produção)
.
Container (Docker): Unidade padrão de software que empacota o código e suas dependências, permitindo que a aplicação rode de forma rápida e confiável em diferentes ambientes
.
Kubernetes (K8s): Plataforma de orquestração de contêineres que automatiza a implantação, o escalonamento e o gerenciamento de aplicações containerizadas
.
Modelo OSI: Arquitetura de rede dividida em sete camadas (Física, Enlace, Rede, Transporte, Sessão, Apresentação e Aplicação) para padronizar as comunicações
.
Pentest: Teste de intrusão realizado para identificar e explorar vulnerabilidades em um sistema, simulando um ataque real para avaliar a segurança
.
Pilares da Segurança: Princípios fundamentais que incluem Confidencialidade, Integridade, Disponibilidade, Autenticidade e Não-repúdio
.
Shift Left Security: Abordagem que busca integrar práticas e testes de segurança o mais cedo possível no ciclo de vida de desenvolvimento de software
.
WAF (Web Application Firewall): Filtro de segurança que protege servidores de aplicações contra ataques específicos da web, como SQL Injection e Cross-Site Scripting (XSS)
.
4. Engenharia de Software e Gestão
BPMN: Notação padrão para modelagem de processos de negócio, permitindo representar graficamente fluxos de trabalho e eventos
.
COBIT 2019: Framework focado na governança e gestão da informação e tecnologia empresarial, que separa claramente os domínios de governança (Avaliar, Dirigir, Monitorar) dos de gestão
.
Design Patterns (GoF): Catálogo de 23 soluções comprovadas para problemas recorrentes, divididos em Criacionais (ex: Singleton), Estruturais (ex: Adapter) e Comportamentais (ex: Memento)
.
Scrum: Metodologia ágil para gestão de projetos composta por ritos (Sprints, Daily Meetings, Review) e papéis (Product Owner, Scrum Master, Team)
.
TDD (Test-Driven Development): Prática de desenvolvimento onde os testes unitários são escritos antes do código funcional da aplicação
.
UML: Linguagem de modelagem unificada usada para especificar, visualizar e documentar artefatos de sistemas de software, com diagramas como os de Classe e Sequência
